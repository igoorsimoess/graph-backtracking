\section{O que é Backtracking?}

Backtracking é uma técnica de busca utilizada para encontrar soluções em problemas que possuem múltiplas possibilidades.

Sua principal característica é construir a solução gradualmente. Em vez de tentar resolver o problema de uma única vez, o algoritmo adiciona uma escolha por etapa.

Sempre que uma escolha leva a uma situação inválida, o algoritmo retorna para o estado anterior e tenta outra alternativa.

O processo pode ser resumido em quatro etapas:

\begin{passos}[Quatro etapas]
\begin{enumerate}[leftmargin=1.4em]
  \item Fazer uma escolha.
  \item Verificar se a escolha continua válida.
  \item Avançar para a próxima etapa.
  \item Voltar atrás caso seja necessário.
\end{enumerate}
\end{passos}

Esse mesmo procedimento pode ser descrito de forma genérica em pseudocódigo:

\begin{pseudocodigo}[Esqueleto do Backtracking]
resolver(solução\_parcial):\\
\quad se solução\_parcial é completa:\\
\quad\quad registrar solução e retornar verdadeiro\\
\quad para cada escolha possível:\\
\quad\quad se a escolha é válida:\\
\quad\quad\quad aplicar a escolha\\
\quad\quad\quad se resolver(solução\_parcial estendida): retornar verdadeiro\\
\quad\quad\quad desfazer a escolha \textcolor{btPodado}{(retrocesso)}\\
\quad retornar falso
\end{pseudocodigo}

Essa abordagem garante que todas as possibilidades relevantes sejam analisadas. Pode-se pensar no Backtracking como uma busca exaustiva organizada: no pior caso ele ainda é capaz de examinar todas as possibilidades, mas, na prática, evita explorar caminhos que já se sabe que não levarão a uma solução.
